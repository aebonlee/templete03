## 4. 구현 내역
- **페이지**: 홈(히어로+가치3+커리큘럼+가격) · 커리큘럼 · 레슨(잠금/이전·다음) · 로그인
- **컴포넌트**: Header · Footer · Section · Reveal(스크롤 리빌) · Highlighter(시그니처) · Grain(종이결)
- **인증**: `AuthContext` + Supabase OAuth(Google/Kakao), env 미설정 시 graceful fallback
- **DB**: `supabase/schema.sql` — `tpl_profiles / tpl_progress / tpl_notes / tpl_orders` + RLS + 가입 트리거. JS는 `t('progress')` 헬퍼로 접두사 처리

# 3차 — 결제 서버검증 (Edge Function)

## 구현
- `supabase/functions/verify-payment/index.ts` — Deno Edge Function
  - ① 호출자 JWT 검증 → ② 포트원 REST API(`api.iamport.kr`)로 imp_uid 실결제 조회
  - ③ 서버측 `PLAN_PRICES` 와 금액·상태(`paid`)·merchant_uid 대조(위변조 차단)
  - ④ service role 로 `tpl_orders` 확정 + `tpl_profiles.plan='pro'` 승급
- `lib/payment.js` — 결제 성공 후 클라이언트 직접 기록 제거 →
  `supabase.functions.invoke('verify-payment')` 로 **서버검증 통과해야 resolve**
- `supabase/functions/README.md` — 배포·시크릿(IMP_REST_API_KEY/SECRET) 가이드

## 배포 상태
- 함수 코드/클라이언트 연동 커밋·배포 완료 (`npm run build` ✓)
- ⚠ **함수 배포는 사용자 작업**: 제공된 sbp_ 토큰이 401(만료/권한). 본인 유효
  토큰으로 `supabase functions deploy verify-payment` + 포트원 REST 시크릿 등록 필요.
  (그 전까지 결제 시도는 "검증 요청 실패"로 안전하게 막힘)
- 점검 결과: Supabase에 `verify-payment`가 떠 있으나 **레포 코드와 다른 기존 버전**
  (영어 에러·인증 우선검사 없음). 사용자 결정 → 레포 버전으로 재배포(덮어쓰기) 예정.
  단 제공된 sbp_ 토큰이 계속 401이라 배포는 유효 토큰 확보 후 진행.

# 4차 — 대면 강의 + 웹 학습자료 중심 개편

## 배경
강의 형태가 영상 녹화 전달보다 **대면 강의 + 웹 학습자료 제공**이 더 많음.
영상 위주 구성을 자료 중심으로 바꾸되 **영상 제공 기능은 그대로 유지**.

## 변경
- `components/LessonContent.jsx` — 웹 학습자료 블록 렌더러 신설
  (h/p/callout/list/steps/code/image/quote/divider). `callout`은 형광펜 강조 박스
- `config/site.js` — 레슨 모델 확장: `kind('reading'|'video')` + `content` 블록 +
  선택 `videoId`. 샘플을 자료 중심으로 재작성(영상 보충 2개 유지), `lessonFormat()` 헬퍼
- `pages/Lesson.jsx` — 영상은 videoId 있을 때만, 읽기 자료는 항상 렌더(영상+자료 병행)
- `pages/Curriculum.jsx` — 레슨별 형태 태그(▶영상 / ✎자료 / 영상+자료)
- 메타 라벨: 읽기/시청 시간 구분
